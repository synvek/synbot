# 第一步

恭喜您安装了 Synbot！本指南将帮助您开始使用 AI 助手。

## 快速入门清单

1. 鉁?**安装完成** - Synbot 已安装在您的系统上
2. 鉁?**配置创建** - 您有一个可用的 `config.json` 文件
3. 鉁?**API 密钥配置** - LLM 提供商 API 密钥已设置
4. 鉁?**渠道启用** - 至少配置了一个消息渠道
5. 鈴?**第一次交互** - 让我们开始吧！

## 您的第一次对话

### 1. 启动 Synbot

```bash
# 启动 Synbot
synbot start

# 预期输出：
# 2024-01-15 10:30:45 INFO synbot: 启动 Synbot v0.1.0
# 2024-01-15 10:30:45 INFO synbot::channels::telegram: 已连接为 @your_bot_name
# 2024-01-15 10:30:45 INFO synbot: Synbot 成功启动
```

### 2. 打开您的消息应用

- **Telegram**：通过用户名搜索您的机器人
- **Discord**：转到您添加了机器人的服务器
- **Feishu**：打开与您的机器人的聊天

### 3. 发送您的第一条消息

```
您：/start
```

预期响应：
```
机器人：您好！我是您的 Synbot 助手。我可以帮助您完成各种任务，包括文件管理、网络搜索和命令执行。我今天能如何帮助您？
```

### 4. 尝试基本命令

```
您：您能做什么？
```

机器人应该解释其功能和可用工具。

## 要尝试的基本任务

### 1. 文件操作

```
您：您能列出当前目录中的文件吗？
```

机器人将使用 `list_files` 工具显示目录内容。

### 2. 系统信息

```
您：当前日期和时间是什么？
```

机器人将使用 `get_time` 工具提供此信息。

### 3. 简单计算

```
您：计算 15 * 24
```

机器人将使用 `calculate` 工具执行计算。

### 4. 网络搜索（如果已配置）

```
您：搜索关于 Rust 编程的信息
```

如果您配置了 Brave Search API 密钥，机器人将执行网络搜索。

## 理解工作流

### Synbot 如何工作

1. **您通过消息渠道发送消息**
2. **Synbot 接收**消息并处理它
3. **AI 决定**使用哪些工具（如果有）
4. **工具执行**带权限检查
5. **结果合并**到响应中
6. **响应发送**回给您

### 工具执行流程

当您要求 Synbot 执行任务时：

```
用户："列出 /tmp 中的文件"

1. AI 分析请求
2. 确定需要 `list_files` 工具
3. 检查权限（在基本配置中允许）
4. 执行：list_files { "path": "/tmp" }
5. 返回："/tmp 中的文件：file1.txt, file2.log"
6. AI 格式化响应："这是 /tmp 中的文件：..."
```

## 常见首次使用问题

### 1. 机器人无响应

**检查**：
- Synbot 是否正在运行？（`ps aux | grep synbot`）
- 日志中是否有错误？（`tail -f ~/.synbot/logs/synbot.log`）
- 渠道是否正确配置？

### 2. 权限被拒绝

**检查**：
- 查看 `config.json` 中的权限规则
- 检查命令是否需要审批
- 在日志中查找"权限被拒绝"

### 3. 响应缓慢

**检查**：
- LLM API 响应时间
- 网络连接性
- 系统资源使用

### 4. 工具不工作

**检查**：
- 工具是否在配置中启用？
- 是否提供了必需参数？
- 检查工具特定日志

## 首次使用后的后续步骤

- **添加更多允许的命令**到权限规则

- **配置多个渠道**

尝试使用不同的工具：

```bash
# 检查可用工具
curl http://localhost:18888/api/tools  # 如果 Web 仪表板已启用
```

### 4. 设置监控

- 监控响应时间
- 为错误设置警报
- 跟踪使用模式

## 首次使用问题故障排除

### 问题："命令不理解"
**解决方案**：在您的请求中更具体。Synbot 最适合清晰、直接的指令。

### 问题："工具执行失败"
**解决方案**：检查日志中的错误消息。常见问题包括缺少权限或不正确的参数。

### 问题："机器人无响应"
**解决方案**： 
1. 检查 Synbot 是否正在运行
2. 检查渠道连接性
3. 查找网络问题
4. 验证 API 密钥是否有效

### 问题："响应太慢"
**解决方案**：
1. 检查 LLM 提供商状态
2. 减少配置中的 `maxTokens`
3. 使用更快的模型
4. 检查网络延迟
