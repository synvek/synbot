# 快捷命令

Synbot 在对话中支持**快捷命令**（斜杠命令）：输入命令（如 `/clear`）可控制会话、启动工作流或查看状态。命令**不区分大小写**，且按**精确前缀**匹配（消息必须仅为命令本身，或命令后仅跟空白）。例如 `/stop` 会匹配；`/stop 现在` 不会。

## 速查表

| 命令 | 说明 |
|------|------|
| `/workflow <描述>` | 根据任务描述创建并运行新工作流。 |
| `/workflow continue` | 继续当前会话已保存的工作流（暂停或超时后）。 |
| `/workflow` + JSON | 根据 JSON 定义创建工作流；机器人会在运行前请你确认。 |
| `/stop` 或 `/cancel` | 停止当前正在运行的工作流或 agent 任务。 |
| `/resume` | 恢复当前会话的工作流（与 `/workflow continue` 相同）。 |
| `/status` | 查看当前会话信息及工作流状态（若有）。 |
| `/clear` | 清除当前会话（对话历史与工作流状态）。 |

---

## 工作流命令

### `/workflow <描述>`

创建并运行新工作流。消息的其余部分为任务描述；机器人会通过模型生成步骤并执行。

**示例：**

```
/workflow 总结仓库中 top 3 的 issue 并起草一份简短状态报告
```

### `/workflow continue`

继续当前会话已保存的工作流（例如暂停或等待用户输入超时后）。与 `/resume` 效果相同。

### `/workflow` + JSON

可在消息中提供 JSON 工作流定义（代码块或 `{ ... }`）。机器人会解析、展示步骤摘要并在运行前请你确认。步骤类型与 JSON 格式见[工作流指南](./workflow.md)。

## 控制命令

这些命令**随时可用**；当工作流或 agent 任务**正在运行**时，它们是除启动新任务外唯一可用的控制方式。若在运行中发送普通消息，会收到简短提示，列出：`/stop`、`/status`、`/clear`、`/resume`。

### `/stop` 或 `/cancel`

停止当前正在运行的工作流或 agent 任务。工作流状态会以已取消形式保存，你仍可查看或通过 `/clear` 清空会话。

**用法：** 仅发送 `/stop` 或 `/cancel`（可带尾部空格）。不区分大小写。

### `/resume`

恢复当前会话的工作流。与 `/workflow continue` 等价。在暂停（如用户输入超时）后使用，可从最后一步继续。

### `/status`

查看当前会话及工作流状态，例如：

- 会话 key、消息数、是否在运行  
- 工作流 id、状态、当前步骤索引（若存在工作流）

### `/clear`

清除当前会话：该会话的对话历史与工作流状态会被重置（与 `reset_session` 工具一致）。用于在同一频道重新开始。

**用法：** 仅发送 `/clear`（可带尾部空格）。不区分大小写。

---

## 其他命令

### `/skills`

`/skills` **不是**控制命令。该消息会传给模型，模型可根据系统提示中的 **Skills** 部分作答（如列出或说明可用技能）。它不会停止或改变工作流/会话状态。

---

## 小结

| 操作 | 命令 |
|------|------|
| 按描述创建工作流 | `/workflow <你的任务描述>` |
| 按 JSON 创建工作流 | `/workflow` + JSON 块；被询问时确认 |
| 暂停后继续工作流 | `/workflow continue` 或 `/resume` |
| 停止当前运行 | `/stop` 或 `/cancel` |
| 查看会话与工作流状态 | `/status` |
| 清除会话与工作流 | `/clear` |

所有命令均按精确前缀匹配，且仅允许尾部空格。更多工作流说明请参阅[工作流指南](./workflow.md)。
